# Configuration

This guide covers the configuration options available for your Botim Open Platform application.

## Configuration File

The SDK supports configuration via a `botim.config.ts` file in your project root:

```typescript
import { defineConfig } from '@botim/sdk'

export default defineConfig({
  apiKey: process.env.BOTIM_API_KEY,
  apiSecret: process.env.BOTIM_API_SECRET,

  environment: 'sandbox',

  messaging: {
    defaultLanguage: 'en',
    maxMessageLength: 4096,
    enableReadReceipts: true,
  },

  webhooks: {
    url: 'https://your-app.com/api/webhooks/botim',
    secret: process.env.BOTIM_WEBHOOK_SECRET,
    events: ['message.received', 'message.delivered', 'payment.completed'],
  },

  rateLimit: {
    maxRequests: 1000,
    windowMs: 60_000,
  },
})
```

## Environment-Specific Configuration

### Sandbox Environment

The sandbox environment is ideal for development and testing:

| Setting | Value |
|---------|-------|
| API Base URL | `https://sandbox-api.botim-op.com/v3` |
| WebSocket URL | `wss://sandbox-ws.botim-op.com` |
| Rate Limit | 100 requests/minute |
| Message Quota | 10,000/day |

### Production Environment

Production settings for live applications:

| Setting | Value |
|---------|-------|
| API Base URL | `https://api.botim-op.com/v3` |
| WebSocket URL | `wss://ws.botim-op.com` |
| Rate Limit | 10,000 requests/minute |
| Message Quota | Unlimited (based on plan) |


## Webhook Configuration

Webhooks allow your application to receive real-time notifications:

```typescript
// Webhook handler example
export default defineEventHandler(async (event) => {
  const signature = getHeader(event, 'x-botim-signature')
  const body = await readBody(event)

  if (!botim.webhooks.verify(signature, body)) {
    throw createError({ statusCode: 401 })
  }

  switch (body.event) {
    case 'message.received':
      await handleIncomingMessage(body.data)
      break
    case 'payment.completed':
      await handlePayment(body.data)
      break
  }

  return { ok: true }
})
```

## Advanced Options

### Retry Policy

Configure automatic retries for failed requests:

```typescript
export default defineConfig({
  retry: {
    maxAttempts: 3,
    backoffMs: 1000,
    backoffMultiplier: 2,
  },
})
```

### Logging

Enable detailed logging for debugging:

```typescript
export default defineConfig({
  logging: {
    level: 'debug', // 'error' | 'warn' | 'info' | 'debug'
    format: 'json',
  },
})
```
