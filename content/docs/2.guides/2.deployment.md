# Deployment

This guide covers deployment strategies and best practices for running your Botim Open Platform integration in production.

## Deployment Checklist

Before deploying to production, verify the following:

- [ ] Switch environment from `sandbox` to `production`
- [ ] Set all environment variables on the host
- [ ] Configure webhook URLs to use HTTPS
- [ ] Enable rate limiting
- [ ] Set up monitoring and alerting
- [ ] Review security best practices

## Docker Deployment

### Dockerfile

```dockerfile
FROM node:22-alpine AS builder
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN corepack enable && pnpm install --frozen-lockfile
COPY . .
RUN pnpm build

FROM node:22-alpine
WORKDIR /app
COPY --from=builder /app/.output .output
ENV PORT=8080
EXPOSE 8080
CMD ["node", ".output/server/index.mjs"]
```

### Docker Compose

```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - '8080:8080'
    environment:
      - BOTIM_API_KEY=${BOTIM_API_KEY}
      - BOTIM_API_SECRET=${BOTIM_API_SECRET}
      - BOTIM_WEBHOOK_SECRET=${BOTIM_WEBHOOK_SECRET}
    restart: unless-stopped
    healthcheck:
      test: ['CMD', 'curl', '-f', 'http://localhost:8080/api/health']
      interval: 30s
      timeout: 10s
      retries: 3
```

## Cloud Deployments

### AWS (ECS / Fargate)

1. Build and push Docker image to ECR
2. Create ECS task definition with environment variables
3. Configure Application Load Balancer with HTTPS
4. Set up auto-scaling based on CPU/memory usage

### Azure App Service

1. Runtime: Node.js 22 LTS (Linux)
2. Startup command: `node .output/server/index.mjs`
3. Deploy the `.output/` directory
4. Configure environment variables in Application Settings

### Kubernetes

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: botim-integration
spec:
  replicas: 3
  selector:
    matchLabels:
      app: botim-integration
  template:
    metadata:
      labels:
        app: botim-integration
    spec:
      containers:
        - name: app
          image: your-registry/botim-integration:latest
          ports:
            - containerPort: 8080
          env:
            - name: BOTIM_API_KEY
              valueFrom:
                secretKeyRef:
                  name: botim-secrets
                  key: api-key
          livenessProbe:
            httpGet:
              path: /api/health
              port: 8080
            initialDelaySeconds: 10
            periodSeconds: 30
```

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `BOTIM_API_KEY` | Yes | Your API key from the Developer Portal |
| `BOTIM_API_SECRET` | Yes | Your API secret |
| `BOTIM_WEBHOOK_SECRET` | Yes | Secret for verifying webhook signatures |
| `PORT` | No | Server port (default: 3000) |
| `NODE_ENV` | No | Set to `production` for optimizations |

## Security Best Practices

1. **Never commit secrets** — Use environment variables or a secrets manager
2. **Use HTTPS** — All webhook URLs must use HTTPS in production
3. **Validate webhooks** — Always verify webhook signatures
4. **Rate limiting** — Implement rate limiting on your webhook endpoints
5. **Rotate keys** — Regularly rotate API keys and secrets

## Monitoring

### Health Check Endpoint

Implement a health check endpoint for load balancers:

```typescript
export default defineEventHandler(() => {
  return {
    status: 'ok',
    timestamp: new Date().toISOString(),
    version: '1.0.0',
  }
})
```

### Recommended Metrics

- API response times (p50, p95, p99)
- Webhook delivery success rate
- Message throughput (messages/second)
- Error rates by endpoint
- Active WebSocket connections
