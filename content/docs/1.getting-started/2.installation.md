# Installation

This guide walks you through setting up your development environment for building with the Botim Open Platform.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** 22 LTS or later
- **pnpm** (recommended) or npm
- A **Botim Developer Account** — [Sign up here](https://developers.botim.com)

## Step 1: Install the SDK

Install the Botim Open Platform SDK using your preferred package manager:

```bash
# Using pnpm (recommended)
pnpm add @botim/sdk

# Using npm
npm install @botim/sdk

# Using yarn
yarn add @botim/sdk
```

## Step 2: Obtain API Keys

1. Log in to the [Developer Portal](https://developers.botim.com)
2. Navigate to **Settings** > **API Keys**
3. Click **Generate New Key**
4. Copy both the **API Key** and **API Secret**

> **Important:** Never expose your API Secret in client-side code. Always keep it on the server.

## Step 3: Initialize the SDK

Create a configuration file in your project:

```typescript
import { BotimSDK } from '@botim/sdk'

const botim = new BotimSDK({
  apiKey: process.env.BOTIM_API_KEY,
  apiSecret: process.env.BOTIM_API_SECRET,
  environment: 'sandbox', // Use 'production' for live apps
})

export default botim
```

## Step 4: Verify the Setup

Run a quick test to verify everything is working:

```typescript
const status = await botim.health.check()
console.log('Platform status:', status)
// Output: Platform status: { ok: true, version: '3.2.0' }
```

## Environment Variables

Create a `.env` file in your project root:

```bash
BOTIM_API_KEY=your_api_key_here
BOTIM_API_SECRET=your_api_secret_here
BOTIM_WEBHOOK_SECRET=your_webhook_secret
```

## Troubleshooting

### Common Issues

**Authentication Error (401)**
- Verify your API key and secret are correct
- Check that your developer account is active

**Connection Timeout**
- Ensure you're using the correct environment (`sandbox` vs `production`)
- Check your network firewall settings

**SDK Version Mismatch**
- Run `pnpm update @botim/sdk` to get the latest version
